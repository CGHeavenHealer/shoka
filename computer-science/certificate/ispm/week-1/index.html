# 信息系统

# 2 类信息系统生命周期

  1. 立项（系统规划）
  2. 开发（系统分析、系统设计、系统实施）
  3. 运维（运行维护）
  4. 消亡

四阶段（立开云消）；

# 信息系统生命周期产物

  • 规划：可研报告、系统任务书；
  • 分析：系统说明书；
  • 设计：设计说明书；
  • 实施：进展报告、测试分析报告；
  • 运行和维护：系统运行情况；

# 信息系统 4 种开发方法

结构化方法、面向对象方法、原型化方法、面向服务的方法

# 结构化方法

  • 精髓：自顶向下、逐步求精、模块化设计；
  • 优点：体现了逐层分解、逐步求精的原则强调完整和顺序，避免过程混乱；
  • 缺点：开发周期长；难以适应需求变化；很少考虑数据结构；
  • 适用范围：数据处理领域的问题；
  • 主要工具：DFD 图、E-R 图、数据字典、判定树；

# 面向对象方法

  • 优点：更好的复用性、缩短开发周期、利于交流沟通；
  • 缺点：必须依靠一定的 OO 技术支持；紧耦合无法涉足系统分析以前的开发环节；
  • 适用范围：普遍适用于各类系统；
  • 主要工具：UML。

面向对象一些基本概念：
类（事物的抽象，包含属性、操作）、对象（类的实例化）
继承（子类继承父类所有功能，并进行扩展）、封装（隐藏实现细节、对外公开接口）
接口（一种规范，用于和外部实体消息通信）、多态（一个接口，多种实现）

# 原型方法

  • 优点：开发周期短、综合开发效益高、用户满意度增加、利于移交和维护；
  • 缺点：开发的环境要求高、管理水平要求高；
  • 适用范围：需求不明确的、分析层面难度大的系统。
  • 原型的分类
    • 水平原型：未真实实现功能、主要用在界面上、较多应用于需求；
    • 垂直原型：实现了部分功能、主要用在复杂算法上、较多应用于设计；
    • 抛弃式原型：达到预期目的后，原型本身被抛弃；
    • 演化式原型：逐步将原型演化成最终系统；

# 面向服务的方法

松耦合、快速响应、高复用性、高互操作性；

# 每日一练

# 210630

  1. 以下关于信息的质量属性的叙述中，不正确的是

    • 完整性，对实物状态描述中的全面程度
    • 可验证性，信息的来源，采集方法，传输过程是符合预期的
    • 安全性，在信息的生命周期中，信息可以被非授权访问的可能性
    • 经济性，信息获取，传输带来的成本在可以接受的范围之内

    软考高项教程第三版 P3：可验证性 —— 指信息的主要质量属性可以被证实或者证伪的程度。
    选项 B 后面的描述是信息的 “可靠性” 的描述

  2. 信息系统的生命周期中，每个阶段完成后都要向下一阶段交付一定的文档， 是系统规划阶段交付的文档？

    • 系统方案说明书；
    • 系统设计说明书；
    • 用户说明书；
    • 可行性研究报告；

    软考高项教程第三版 P11：信息系统的生命周期可以简化为系统规划、系统分析、系统设计、系统实施、运行维护等阶段。
    其中系统规划阶段会写出可行性研究报告、系统设计任务书。

  3. 在 ISO OSI/RM 中， 实现数据压缩功能？

    • 应用层
    • 表示层
    • 会话层
    • 网络层

    软考高项教程第三版 P18：表示层管理数据的加密、数据转换、格式化和文本压缩。

  4. 在国家信息化体系六要素中， 是国家信息化的核心任务，是国家信息化建设取得实效的关键？

    • 信息技术和产业；
    • 信息资源的开发和利用；
    • 信息人才；
    • 信息化政策法规和标准规范；

    软考高项教程第三版 P9：国家信息化体系的六要素是：信息技术应用（龙头）、信息技术和产业（物质基础）、信息化人才（成功之本）、信息化政策法规和标准规范（保障）、信息资源（核心）、信息网络（设施）。

  5. 关于 TCP 和 UDP 地说法， 是错误的？

    • TCP 和 UDP 都是传输层的协议
    • TCP 是面向连接的传输协议
    • UDP 是可靠的传输协议
    • TCP 和 UPD 都是以 IP 协议为基础的

    软考高项教程第三版 P20：UDP 是一种不可靠的、无连接的协议。
    TCP 是可靠的、面向连接的，所以 B 说法正确；
    他们 2 个都是传输层的协议，所以 A 选项正确；
    IP 协议属于三层网络层，四层传输层的 TCP 和 UDP 都要以三层网络层的 IP 协议为基础，所以 D 选项正确
