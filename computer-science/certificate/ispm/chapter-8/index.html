# 质量管理大师

# 戴明

戴明 PDCA 环
  • P 计划：设定为了达到目标所必须的方法或标准
  • D 实施（执行）：按计划逐步实施具体工作
  • C 确认（检查）：确认并检查实施的效果
  • A 措施（行动）：确认实际效果与计划差异，并根据需要采取措施

# 现代质量管理的重要理念

  1. 客户满意
    • 了解、评估、定义和管理要求。
    • 符合要求 + 适合使用
  2. 预防胜于检查
    • 事先制定周密、科学的计划来预防质量问题产生。
    • 质量是规划和实施出来的，不是检查出来的。
  3. 持续改进或凯思恩 Kaizen
    • 通过持续不断的小改进积累成大改进，往往比瞬间的大改进更有价值。
    • PDCA 环是质量改进的基础。
    • 常用的过程改进模型包括：马尔科姆波多里奇模型、OPM3 和 CMMI。
  4. 管理层的责任
    • 85/15 原则。
    • 管理者对质量负 85% 的责任，工人只有 15% 的责任。
  5. 质量成本 COQ
    • 一致性工作成本 + 非一致性工作成本；
    • 一致性工作是为预防工作出错而做的附加努力；
    • 非一致性工作是为纠正已经出现的错误而做的附加努力。
  6. 反对 “镀金”
    • 镀金不会增加项目价值，会造成较大机会成本；
    • 资源有限，只提供答应的东西。
  7. 第一次就把事情做对 质量是免费的
    • 就是 “零缺陷管理”，最节约成本的方法，虽不能绝对做到，但可不断接近。
  8. 准时制（零库存）管理 JIT
    • 由于零库存，没有多余的材料，促使人们更注重质量，力争一次做对，力争零缺陷
  9. 全面质量管理 TQM
    • 强调全过程的质量管理
    • 强调全员参与质量管理

# 质量

质量 Qualiy
  • 反映实体满足主体明确和隐含需求的能力的特性总和。 ISO的定义
  • 一组固有特性满足要求的程度。 国家标准GB/T 19000-2008的定义
项目质量
工作质量 + 产品质量。
项目的质量是应顾客的要求进行的，不同的顾客有着不同的质量要求，其意图已反映在项目合同中。
因此，项目合同通常是进行项目质量管理的主要依据。

# 质量与等级的区别

质量
作为实现的性能或成果，是一系列内在特性满足要求的程度。 ISO 9000的定义
等级
作为设计意图，是对用途相同但技术特性不同的可交付成果的级别分类。
  • 高等级并不意味着一定高质量；
  • 低等级也并不意味着一定低质量；
  • 质量水平未达到质量要求肯定是个问题，而低等级不一定是个问题。
  • 项目经理及项目管理团队负责权衡，以便同时达到所要求的质量与等级水平。

# 质量管理标准体系

  • ISO 9000 质量管理的 8 项基本原则
    1. 领导作用
    2. 全员参与
    3. 过程方法
    4. 持续改进
    5. 以顾客为中心
    6. 与供方互利的关系
    7. 管理的系统方法
    8. 基于事实的决策方法

领导全员用系统和事实的方法，对于顾客和供应商，进行过程的持续改进

全面质量管理 TQM
  • 一种全员、全过程、全企业的品质管理。
  • 是一个组织以质量为中心，以全员参与为基础，通过让顾客满意和本组织所有成员及社会收益而达到永续经营的目的。
  • 它注重顾客需要，强调参与团队工作，并力争形成一种文化。
  • 组成的 4 个要素：
    1. 结构
    2. 技术
    3. 人员
    4. 变革推动者
6σ 方法
  • 核心是将所有的工作作为一种流程，采用量化的方法分析流程中影响质量的因素，找出最关键的因素通过 DMAIC （确定、测量、分析、改进、控制）方法加以改进从而达到更高的客户满意度。
  • 优越之处是：从项目实施过程中改进和保证质量，减少了检控质量的步骤，降低成本，并培养了质量意识。
  • 在质量上表示为：每百万不合格率少于 3.4。

# 项目质量管理

项目质量管理 Quality Management
确定质量方针、质量目标和职责，并通过质量体系中的质量规划、质量保证和质量控制以及质量改进来使其实现所有管理职能的全部活动。
质量方针
由组织的最高管理者正式发布的该组织总的质量宗旨和方向，体现了组织的质量意识和质量追求。
质量目标
在质量方面所追求的目的，是落实质量方针的具体要求。必须明确、具体、量化。
  • 项目质量管理需要兼顾项目管理与项目可交付成果两个方面。
  • 质量管理，更重要的是价值观（理念）问题，而不是技术问题。
启动过程组规划过程组执行过程组监控过程组收尾过程组
规划质量管理 QP
识别项目及其可交付成果的质量要求和标准，书面描述项目将如何证明符合质量要求和标准 定指南，定指标
实施质量保证 QA
审计质量要求和质量控制测量结果，确保采用合理的质量标准
控制质量 QC
监督并记录质量活动执行结果，以便评估绩效，并推荐必要的变更

# 规划 规划质量管理

  • 识别项目及其可交付成果的质量要求和 / 或标准，并书面描述项目将如何证明符合质量要求的过程。
本过程的作用
为整个项目中如何管理和确认质量提供指南和方向。
